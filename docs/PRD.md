# PRD (Product Requirements Document)

**프로젝트명:** Tika  
**버전:** 1.0 (MVP)  
**작성일:** 2026-06-10

---

## 1. 제품 개요

Tika는 티켓 기반 칸반 보드 TODO 앱이다.  
할 일을 티켓 단위로 등록하고, 드래그 앤 드롭으로 진행 상태를 관리한다.  
단일 사용자를 대상으로 하며, 빠른 업무 흐름 파악과 일정 추적에 초점을 맞춘다.

---

## 2. MVP 범위

### 포함
- 단일 사용자 환경 (인증 없음)
- 고정 4컬럼 구조: **Backlog / TODO / In Progress / Done**
- 티켓 CRUD (생성·조회·수정·삭제)
- 드래그 앤 드롭을 통한 컬럼 간 상태 이동
- 티켓별 우선순위 관리: LOW / MEDIUM / HIGH
- 티켓별 날짜 관리
  - 사용자 입력: 계획시작일(`plannedStartDate`), 종료예정일(`dueDate`)
  - 자동 설정: 시작일(`startedAt`, TODO 이동 시), 종료일(`completedAt`, DONE 이동 시)
- 오버듀 판정: 종료예정일 초과 + 미완료 티켓 시각적 표시

### 제외 (2차 스펙)
| 기능 | 사유 |
|------|------|
| 사용자 인증 / 멀티사용자 | MVP는 단일 사용자 전제 |
| 커스텀 컬럼 추가/수정 | 고정 4컬럼으로 충분 |
| 라벨 / 코멘트 | 협업 기능으로 분류 |
| 파일 업로드 | 인프라 복잡도 증가 |

---

## 3. 사용자 시나리오

```
1. 할 일 등록
   └─ Backlog에 티켓 생성 (제목, 설명, 날짜 입력)

2. 상태 이동
   └─ 드래그 앤 드롭으로 Backlog → TODO → In Progress → Done

3. 수정 / 삭제
   └─ 티켓 클릭 → 상세 모달에서 내용 수정 또는 삭제

4. 완료 확인
   └─ Done 컬럼에서 완료된 티켓 목록 확인
      실제 시작일 / 종료일과 계획일 비교
```

## 4. 핵심 기능 목록

REQUIREMENTS.md의 FR-001~008과 대응한다.

| FR ID | 기능명 | 설명 |
|-------|--------|------|
| FR-001 | 티켓 생성 | 제목(필수), 설명, 우선순위, 계획시작일, 종료예정일을 입력해 새 티켓 생성. 생성 시 기본 컬럼은 Backlog |
| FR-002 | 티켓 목록 조회 | 칸반 보드에서 전체 티켓을 컬럼별로 표시. Backlog는 사이드바로 분리. `isOverdue` 파생 필드 포함 |
| FR-003 | 티켓 상세 조회 | 티켓 클릭 시 모달로 전체 필드(제목, 설명, 우선순위, 날짜, 상태) 표시 |
| FR-004 | 티켓 수정 | 모달 내에서 제목, 설명, 우선순위, 계획시작일, 종료예정일 편집 후 저장 |
| FR-005 | 티켓 완료 처리 | Done 이동 시 `completedAt` 자동 기록. 되돌릴 시 초기화. `PATCH /api/tickets/:id/complete` |
| FR-006 | 티켓 삭제 | 모달에서 삭제. 삭제 전 확인 다이얼로그 표시. 하드 삭제 |
| FR-007 | 상태/순서 변경 | 드래그앤드롭으로 컬럼 간 이동 및 같은 컬럼 내 순서 변경. `PATCH /api/tickets/reorder` |
| FR-008 | 오버듀 판정 | `dueDate < 오늘 AND status ≠ DONE` 조건의 파생 필드. 조회 시 계산, DB 저장 안 함 |

## 5. 기술 스택 요약

| 분류 | 기술 | 선정 이유 |
|------|------|-----------|
| Framework | Next.js 15 (App Router) | 프론트·백엔드 단일 레포, Route Handler로 API 내장 |
| Language | TypeScript (strict) | 컴파일 타임 타입 안전성, 리팩토링 용이 |
| Frontend | React 19 | 최신 훅·동시성 기능, Next.js와 자연스러운 통합 |
| Styling | Tailwind CSS 4 | 유틸리티 클래스로 빠른 UI 개발, 번들 크기 최소화 |
| Drag & Drop | @dnd-kit/core + sortable | 접근성 지원, 커스텀 센서·전략 조합 가능 |
| ORM | Drizzle ORM | TypeScript-first, 스키마 = 타입 소스, 마이그레이션 단순 |
| DB | Vercel Postgres (Neon) | Vercel 배포 환경과 완벽 통합, 서버리스 연결 풀링 |
| Validation | Zod | 런타임 검증 + 타입 추론 동시 처리, 프론트·백 스키마 공유 |
| Testing | Jest + React Testing Library | 단위·통합 테스트 표준 스택, TDD 사이클 지원 |
| Deployment | Vercel | Next.js 네이티브 배포, Edge/Serverless 자동 최적화 |

```
│  └──────────┘    │  └──────────┘  └──────────┘  └──────────┘  │

│  [+ 티켓 추가]   │                                              │
│                  │                                              │
└──────────────────┴──────────────────────────────────────────────┘
  ← 사이드바 →      ←──────────── 3컬럼 보드 (Swimlane) ──────────→
```

**레이아웃 설명**
- **Backlog 사이드바**: 좌측 고정 패널. 아직 작업에 투입되지 않은 티켓 보관
- **3컬럼 보드**: 우측 메인 영역. TODO → In Progress → Done 순서로 Swimlane 배치
- 컬럼 간 드래그 앤 드롭으로 Backlog ↔ 보드 간 이동 포함
